# vision/ — el modelo estructural a futuro

El **destino**: un CreditOp donde los comercios se adaptan a un estándar (catálogo de productos, política por niveles con herencia, responsabilidades separadas) en vez de que CreditOp se adapte a cada comercio.

- No es un plan de sprint: es el **enfoque robusto** al que deben apuntar las [`../mejoras/`](../mejoras/).
- Está **demostrado en el prototipo** (`playground/flow`: catálogo, niveles, herencia con override).
- Evidencia de las fallas que ataca: apéndice de [`UNIFICACION-Y-RESPONSABILIDADES.md`](./UNIFICACION-Y-RESPONSABILIDADES.md) y [`../codigo/`](../codigo/).
