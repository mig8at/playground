Calculadora por lender (fórmulas en BD) → procesador seguro

Cada lender guarda su fórmula en una columna json lenders.calculator. El backend la pasa por un procesador sandbox (en el server real: App\Support\FormulaCalculator, sin eval) — acá replicado en JS con un parser de aritmética. Elegí un lender de ejemplo o editá cualquier cosa; recalcula solo.
Default = null → identidad: si el lender no tiene calculadora, devuelve lo que llega tal cual (todos los credit). Solo las entidades editadas transforman el monto (ej. renting = Motai). Todas las keys en inglés.

¿Por qué es seguro? (no es eval)

El procesador es un parser de aritmética: solo entiende números, las variables del scope (params + input + resultados previos) y los operadores + - * / ** ( ). Cualquier otra cosa la rechaza. Probá pegar algo "malicioso" en la fórmula de amount y mirá el error:

En el back es igual: solo escalares como variables, sin registrar funciones → aritmética pura, sin ejecución de código. El número autoritativo lo calcula el back y manda el objeto calculated; el front solo lo muestra. Se aplica por lender (cada uno resuelve su calculator) → una entidad nueva = una fila de config, sin código ni deploy.
